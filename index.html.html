🔗 Integração SharePoint (Sincronização Automática)

Configure a sincronização automática com SharePoint para trabalho colaborativo em tempo real.

📌 Uso no OneDrive / SharePoint

Sistema 100% offline — sem servidor necessário.
1. Salve CDO_Sistema_Disponibilidade.html em pasta compartilhada no OneDrive ou SharePoint.
2. Abra com Chrome ou Edge (duplo clique).
3. Dados ficam no localStorage daquela máquina.
4. Para sincronizar: Exportar JSON → salve na pasta compartilhada → nos outros PCs clique Importar JSON.
5. Recomendação: faça exportação diária do JSON para a pasta de backup no OneDrive.

🚀 Fluxo Recomendado (dia a dia)

1. Acesse Registro e lance cada evento de parada assim que ocorrer.
2. Em Histórico CDO, revise dados, filtre por planta/causa e corrija com edição quando necessário.
3. Em Disponibilidade, acompanhe DM/UO/DF por mês e por planta.
4. Em Gatilhos, trate alertas automáticos e chamados manuais.
5. No fechamento do turno/dia, gere backup em Exportar JSON e relatório em Relatórios.

✅ Boas Práticas de Qualidade de Dados

• Sempre preencher Detalhamento com causa observada + efeito no processo.
• Evitar nomes diferentes para o mesmo equipamento (ex: “Correia 1201” e “CR-1201”).
• Usar Reclassificação apenas quando houver evidência técnica posterior.
• Em paradas longas (> 60 min), abrir/acompanhar Gatilho com responsável e prazo.
• No fim do expediente, exportar JSON para backup e rastreabilidade.

🧮 Lógica dos Cálculos Aplicados
Definições matemáticas usadas no sistema com exemplos numéricos práticos

📐 Fórmulas de Disponibilidade (DM, UO e DF)

IndicadorFórmulaDefinição
TC (Tempo Calendário)TC = dias × hCal × 60Tempo total disponível no período em minutos
Manutenção (min)Σ paradas de causa Elétrica + MecânicaBase para cálculo de DM
Total de Paradas (min)Σ todas as paradasInclui causas operacionais, matéria-prima e externas
TDTD = TC − ManutençãoTempo disponível sem perdas de manutenção
TOTO = TC − Total de ParadasTempo operacional efetivo
DMDM = TD / TCDisponibilidade de Manutenção
UOUO = TO / TCUtilização Operacional
DFDF = TO / TDDisponibilidade Funcional

🔢 Exemplo Numérico Completo

Premissas do exemplo:
• Período com 3 dias
• Horas calendário/dia (hCal) = 24
• Paradas Elétrica + Mecânica = 180 min
• Total de paradas (todas as causas) = 300 min

Cálculo:
• TC = 3 × 24 × 60 = 4320 min
• TD = 4320 − 180 = 4140 min
• TO = 4320 − 300 = 4020 min
• DM = 4140 / 4320 = 95,83%
• UO = 4020 / 4320 = 93,06%
• DF = 4020 / 4140 = 97,10%

⏱ Cálculo de Tempo de Parada

O campo Total (min) é calculado automaticamente por:
TotalMin = DataHoraFinal − DataHoraInicial

Exemplo:
• Início: 2026-04-27 10:15
• Fim: 2026-04-27 11:02
• Diferença: 47 min

Regra aplicada no sistema:
• Se diferença < 0, o valor não é aceito (evita tempo inválido).
